# RISCV OS プロジェクト概要

## 目標

RP2350（RISC-Vモード）上で動作する小規模OSと独自コンパイラを作り、
**OS全体をそのOS上でコンパイルできる自己ホスト環境**を実現する。

## ハードウェア

- **ボード**: Raspberry Pi Pico 2（RP2350）
- **CPUモード**: RISC-V（Hazard3コア）のみ使用
- **構成**: 初期はシングルコア
- **メモリ**: SRAM 520KB、Flash 2MB（外部QSPIフラッシュ）

## コンポーネント一覧

### OS

| コンポーネント | 概要 |
|---|---|
| カーネル | 小規模モノリシックカーネル |
| メモリ管理 | 静的アロケータ＋シンプルなヒープ |
| プロセス管理 | シンプルなタスクスケジューラ |
| ファイルシステム | 独自フラットFS |
| 標準Cライブラリ | 最小限のlibc実装 |
| シェル | コマンドラインインタフェース |
| ローダ | 実行ファイルをメモリにロードして実行 |

### コンパイラツールチェーン

| ツール | 概要 |
|---|---|
| `parse` | ソース → AST テキスト |
| `typecheck` | AST → 型付きAST テキスト（型検査・オーバーロード解決） |
| `codegen` | 型付きAST → スタックベースバイトコード テキスト |
| `bcrun` | バイトコードインタプリタ（リファレンス実装、常にテスト維持） |
| `bc2asm` | バイトコード → RISC-V アセンブリ |
| `as` | RISC-V アセンブリ → オブジェクトファイル（GAS互換） |
| `ld` | オブジェクトファイル → 実行ファイル（GNU ld互換） |

## 自己ホストのロードマップ

```
フェーズ0: 言語仕様確定・環境整備（QEMU + 実機）

フェーズ1: インタプリタ（C実装）
           parse / typecheck / interp をCで実装

フェーズ2: コンパイラツールチェーン（独自言語で実装、インタプリタ上で動作）
           parse / typecheck / codegen / bcrun / bc2asm / as / ld を独自言語で実装
           コンパイラで自分自身をコンパイル（自己コンパイル達成）

フェーズ3〜6: OS本体（カーネル・プロセス管理・FS・ユーザランド）をQEMUで開発

フェーズ7: ネイティブコンパイラをOS上で動かす

フェーズ8: OS全体を独自言語で記述し、OS上でビルド（自己ホスト達成）
```
